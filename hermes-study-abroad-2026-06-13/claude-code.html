Claude Code Setup

워크스페이스 안의 .claude/ 및 CLAUDE.md 구성 요약. 사용자가 말한 “with calude”에 해당하는 Claude Code 연동 폴더로 보임.

← 전체 문서

확인 결과

  • with calude라는 이름의 폴더는 없음.
  • 대신 .claude/ 폴더와 CLAUDE.md가 있음.
  • .claude/는 2026-06-12 23:29~23:30 KST에 생성/갱신된 Claude Code 설정 및 slash command 묶음.
  • .claude/settings.local.json은 2026-06-13 00:18 KST에 갱신됨.

CLAUDE.md 역할

Claude Code가 이 워크스페이스에서 Hermes처럼 동작하도록 하는 운영 지침이다. 기본 언어, 날짜 표기, 우선 도메인, 세션 시작 루틴, 파일 라우팅, 안전 원칙, 출력 형식, 투자 비협상 원칙을 정의한다.

.claude/settings.json

Claude Code에서 허용/차단할 명령을 정의한다.

  • 허용: ls, grep, rg, wc, git status, git log, git diff, git branch
  • 차단: sudo, rm -rf, git reset --hard, git clean, force push, .env/.pem/.key/.p12 읽기

.claude/settings.local.json

로컬 전용 허용 규칙이다. Hermes Agent 문서 도메인 WebFetch와 특정 로컬 HTML을 textutil로 읽는 권한이 추가되어 있다. 민감 경로나 원문 내용은 이 웹 페이지에 원문 전체로 싣지 않는다.

Slash Commands

Command역할
/startSTATUS, inbox, project next actions를 보고 세션 동기화 브리핑
/inbox00_Inbox 항목을 Task / Project update / Research / Decision 등으로 분류하고 반영
/daily일간 리포트 생성 및 STATUS 갱신
/weekly주간 리포트 생성 및 다음 주 우선순위 정렬
/decision <주제>의사결정 보조 후 decision note와 프로젝트 Decisions 표 갱신
/research <질문>로컬 우선 리서치 후 출처 포함 research note 생성
/invest <종목/ETF>투자 thesis 노트 생성/갱신. 매수/매도 지시 금지

이번 유학 페이지와의 관계

Claude Code 구성 자체는 유학 결론을 바꾸는 자료는 아니지만, 이 워크스페이스가 Claude Code/Hermes/Codex 사이에서 같은 규칙으로 작동하도록 만든 운영 인프라다. 따라서 웹 페이지에는 원문 전체가 아니라 안전 요약으로만 반영했다.
