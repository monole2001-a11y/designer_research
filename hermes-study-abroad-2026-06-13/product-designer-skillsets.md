# Product Designer 공고 요구 스킬셋 정리 — 2026-06-14

성격: 공개 영문 공고/직무 설명 기반의 의사결정·커리어 준비용 요약. 개인 식별정보나 회사 내부자료 미포함.

## 핵심 결론

미국/글로벌 Product Designer 공고에서 요구하는 역량은 단순 UI 제작보다 **제품 문제 정의 → 리서치/데이터 → 플로우/프로토타입 → 시스템화 → 엔지니어 협업 → 임팩트 커뮤니케이션**으로 수렴한다.

특히 사용자 현재 맥락에서는 “일반 Product Designer”보다 아래 포지셔닝이 더 맞다.

> **AI-aware Senior Product Designer / Design Technologist / UX Engineer-adjacent Product Designer**

이유:
- 삼성 경력은 junior가 아니라 senior/complex-product 포지션의 원재료다.
- SVA IxD + 국비 $100K는 “미국식 공개 포트폴리오와 네트워크”를 만드는 장치로 써야 ROI가 난다.
- H-1B/취업 관점에서는 저임금 generalist보다 **고임금·전문화·시니어 포지션**이 유리하다.

## 1. 공고에서 반복되는 8대 스킬셋

| 스킬셋 | 공고 표현 | 포트폴리오에서 보여줄 증거 |
| --- | --- | --- |
| Product thinking | business goals, product strategy, measurable impact, roadmap | 문제를 왜 풀었는지, 성공 지표가 무엇인지, 결과가 무엇이었는지 |
| UX / interaction design | user flows, IA, wireframes, interaction flows, intuitive experience | 복잡한 문제를 간단한 flow로 바꾼 before/after |
| Visual/UI craft | high-fidelity mockups, typography, layout, visual systems | 완성도 높은 화면, 컴포넌트, 상태값, 반응형 처리 |
| Research & validation | user research, usability testing, customer feedback, market analysis | 인터뷰/테스트가 실제 디자인 결정을 바꾼 사례 |
| Prototyping | interactive prototypes, click-throughs, advanced prototyping | Figma prototype, coded prototype, AI/LLM prototype |
| Design systems | scalable components, patterns, UI specs, consistency | 컴포넌트 구조, 토큰, 사용 규칙, 엔지니어 handoff |
| Cross-functional collaboration | PM/engineering/research collaboration, stakeholder communication | PM/개발자와 trade-off 조정한 의사결정 기록 |
| Communication / rationale | present design rationale, advocate for design, write clearly | 선택 이유, 반대안, 측정 근거를 명확히 설명하는 case study |

## 2. 레벨별 요구 차이

### Early / junior-leaning UX Designer

Google/YouTube early UX 공고와 Amazon UX internship류에서 반복되는 기준:

- 포트폴리오 링크 필수
- 1년 안팎 interaction/product/UX 경험 또는 학업 기반 실무 경험
- Figma로 user flow, high-fidelity mockup, interactive prototype 제작
- 사용자 중심 디자인 방법론 이해
- PM/Engineering과 협업
- ambiguity를 견디고 피드백을 받아 iteration

사용자에게의 함의:
- 이 레벨로 들어가면 “미국 첫 직장”은 가능할 수 있지만, 삼성 경력을 충분히 활용하지 못하고 wage/visa 측면에서 손해일 수 있다.

### Mid/Senior Product Designer

Digital Waffle, Meta 계열 공고 요약, KOHO 등에서 반복되는 기준:

- 3~6년 이상 product/digital design 경험
- end-to-end feature/product ownership
- web/mobile product portfolio
- user research와 usability testing 경험
- Figma 등 modern design tool 숙련
- design systems, accessibility, components 이해
- business goals와 user needs 균형
- agile/cross-functional 환경에서 PM/engineer/stakeholder 협업
- 디자인 rationale을 다양한 audience에게 설명

사용자에게의 함의:
- 삼성 경력은 이 구간을 노려야 한다.
- 단, 삼성 경험이 NDA 때문에 보이지 않으면 senior signal이 약해진다. 그래서 공개 가능한 case 1개가 G5 핵심이다.

### Specialized / high-complexity Product Designer

Harmattan AI류 복잡계 공고와 Design Technologist류 공고에서 반복되는 기준:

- 7~10년 또는 그에 준하는 복잡한 시스템 UX 경험
- data-heavy / mission-critical / healthcare / defense / trading / enterprise SaaS 등 복잡 도메인
- workflow simplification
- design systems + accessibility
- engineering과 fidelity/feasibility 검증
- geospatial, dashboard, real-time telemetry, AI-powered product 등 기술 맥락 이해
- portfolio가 “복잡성을 단순화한 증거”를 보여야 함

사용자에게의 함의:
- “삼성 내부 복잡한 시스템/제품 경험 + AI/개발 역량 + IxD”를 잘 번역하면 일반 PD보다 이쪽이 더 차별적이다.

## 3. AI-era Product Designer에서 추가로 보이는 요구

최근 공고/직무 설명에서 새로 강해진 항목:

| 요구 | 의미 | 사용자 준비 과제 |
| --- | --- | --- |
| GenAI workflow | AI 도구로 ideation, synthesis, prototype 속도 향상 | Claude/Figma/프로토타입 workflow를 case study에 명시 |
| AI product understanding | AI-powered product의 불확실성, trust, transparency 설계 | AI interaction 원칙 정리 + 작은 prototype |
| Technical literacy | LLM/API, frontend, data-heavy UI 이해 | React/TS 또는 coded prototype 증거 |
| Human judgment | AI 산출물 검증, 큐레이션, 의사결정 | “AI를 썼다”가 아니라 “무엇을 검증했고 무엇을 버렸는가” 기록 |

핵심: AI는 “툴 사용”만으로는 약하다. 공고가 원하는 것은 **AI로 더 빨리 그리는 사람**보다 **AI가 들어간 제품을 믿을 수 있게 설계하는 사람**에 가깝다.

## 4. 사용자 포트폴리오로 번역할 체크리스트

첫 공개 case study는 아래 10개 질문에 답해야 한다.

1. 어떤 사용자/비즈니스 문제였나?
2. 문제를 어떻게 발견했나? 리서치/데이터/현장 관찰 근거는?
3. 내가 직접 소유한 범위는 어디까지인가?
4. 초기 flow는 왜 부족했나?
5. 어떤 대안을 비교했고, 왜 버렸나?
6. 최종 flow/UI는 어떻게 복잡성을 줄였나?
7. PM/Engineer/Stakeholder와 어떤 trade-off를 조정했나?
8. 결과를 어떤 지표나 정성 신호로 확인했나?
9. design system/component 관점에서 재사용 가능한 것은 무엇인가?
10. AI/개발/프로토타이핑을 어디에 사용했고, 인간 판단은 어디에 들어갔나?

## 5. SVA 진학 시 2년 안에 만들어야 할 증거

| 시점 | 만들어야 할 증거 | 이유 |
| --- | --- | --- |
| 입학 전~1학기 | 공개 case 1개 착수 | G5 통과, 자기서사 시작 |
| 1학기 | research-to-prototype case | SVA를 단순 학위가 아니라 미국식 case 생산 장치로 쓰기 |
| 2학기 | AI interaction / design technologist prototype | 차별화 포지션 확보 |
| 여름 | internship / CPT / alumni referral 기반 실무 검증 | G4 취업 base rate 검증 |
| 2년차 | portfolio 3개 + design system/AI/비즈니스 임팩트 narrative | senior-ish product designer로 지원 |

## 6. 현재 결정 게이트와의 연결

- **G4 취업 증거**: alumni 인터뷰 질문에 “첫 직장 공고에서 요구한 스킬셋”, “포트폴리오에서 먹힌 case”, “Figma/AI/코드/리서치 중 뭐가 결정적이었는지”를 추가해야 한다.
- **G5 자기서사**: 공개 포트폴리오 1개는 “예쁜 UI”가 아니라 **business impact + ownership + research/validation + technical/AI literacy**를 보여줘야 한다.
- **G3 재정**: senior/전문화 포지셔닝은 단순 커리어 욕심이 아니라, 졸업 후 buffer와 비자/wage 전략에도 연결된다.
